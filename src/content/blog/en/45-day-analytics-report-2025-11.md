# jangwook.net 45-Day Growth Report: What the Data Really Says

> <strong>Transparency Declaration</strong>: This report candidly shares the actual data and lessons learned from 45 days post-launch. We show the real growth journey, not just impressive numbers.

## 1. Executive Summary: The 45-Day Journey

### 1.1 Key Metrics at a Glance

<strong>Analysis Period</strong>: October 7 〜 November 20, 2025 (45 days)
<strong>Previous Report</strong>: [Blog Launch Analysis Report](/en/blog/en/blog-launch-analysis-report) (October 6, real-time data only)

| Key Metric | Actual Performance | 1-Month Target | Achievement | Status |
|-----------|------------------|--------------|------------|--------|
| <strong>Daily Active Users (DAU)</strong> | 16.7/day | 20〜30 | 67〜84% | 🟡 Below Target |
| <strong>Monthly Pageviews</strong> | 〜1,000 | 500〜800 | 125〜200% | 🟢 Exceeded |
| <strong>Organic Search Ratio</strong> | 44.3% | 30%+ | 148% | 🟢 Significantly Exceeded |
| <strong>Returning Visitor Rate</strong> | 13.5% | 10〜15% | 90〜135% | 🟢 Normal Range |

### 1.2 Overall Assessment: 🟡 Mixed Results

<strong>750 users visited the blog.</strong> However, this number hides important context:

<strong>🟢 Success Factors:</strong>
- <strong>Organic search 44.3%</strong> - 2〜3x the average for new blogs (10〜20%)
- <strong>SEO optimization ROI proven</strong> - Daily traffic grew 100% from 5〜10 to 15〜20 users
- <strong>High-quality content validated</strong> - Google Gemini RAG tutorial: 127 pageviews, 31% bounce rate
- <strong>AI tool recommendations</strong> - Natural inflow from Perplexity.ai, ChatGPT, Gemini
- <strong>13.5% returning rate</strong> - Content value proven

<strong>🔴 Challenges to Address:</strong>
- <strong>Nov 17〜20 traffic spike (37%)</strong> - Sustainability verification needed
- <strong>English homepage 91% bounce rate</strong> - Serious UX problem
- <strong>DAU 16.7</strong> - Below 1-month target (20〜30 users)
- <strong>Limited content portfolio</strong> - Traffic concentrated on 2〜3 posts
- <strong>China traffic 19.2%</strong> - Potential bot traffic investigation needed

## 2. Data Analysis: The Story Numbers Tell

### 2.1 Traffic Pattern: 3-Phase Growth Curve

The 45-day traffic evolution can be divided into three phases:

```
Phase 1 (10/7〜10/31): Baseline Formation
└─ 5〜10 users/day average
└─ Initial organic search ranking
└─ Estimated total: 150〜250 users

Phase 2 (11/1〜11/16): SEO Effects Emerge
└─ 15〜20 users/day average (+100〜150% growth)
└─ Organic search dominant
└─ Estimated total: 240〜320 users

Phase 3 (11/17〜11/20): Spike Occurs
└─ 68.5 users/day average (+300〜400% surge)
└─ 274 users in 4 days (37% of total)
└─ Cause unconfirmed - investigation required
```

<strong>Core Insight</strong>: The average DAU of 16.7 is inflated by the Phase 3 spike. <strong>The true baseline is 12〜15 users</strong> (median) for accurate assessment.

## 4. 7 Core Insights

### Insight #1: Traffic Spike Truth - The 37% Mystery

<strong>Data:</strong>
- Nov 17〜20 (4 days) = 274 users (37% of 750 total)
- Average 68.5/day vs. general baseline 12〜15/day
- Remaining 41 days = 476 users (average 11.6/day)

<strong>Meaning:</strong>
The 45-day average DAU of 16.7 is an <strong>illusion</strong>. Excluding the 4-day spike, the actual baseline is <strong>11〜12/day</strong>, which is 45〜60% of the 1-month target (20〜30).

<strong>Hypotheses:</strong>
1. <strong>External mention (60〜70% likelihood)</strong>: Reddit, Hacker News, Korean dev community mention?
2. <strong>Algorithm change (20〜30% likelihood)</strong>: Google ranking algorithm update?
3. <strong>Content viral (5〜10% likelihood)</strong>: Specific post rapid spread?
4. <strong>Bot attack (10〜20% likelihood)</strong>: Related to China traffic?

<strong>Urgent Actions:</strong>
- [ ] Check GA4 Acquisition report for Nov 17〜20
- [ ] Search for blog URL on Reddit, Hacker News, Okky, 44BITS
- [ ] Check Google Search Console keyword ranking changes
- [ ] Collect 2〜3 weeks more data to establish <strong>true baseline</strong>

## 6. Transparency and Learning

### 6.1 Limitations of This Report

<strong>Data Quality:</strong>
- 45-day sample insufficient for seasonal pattern detection
- Nov 17〜20 spike distorts average values
- China traffic legitimacy unverified (19.2% potential overestimate)

<strong>Analysis Constraints:</strong>
- Average session time data absent
- Pages per session unclear
- Improvement effects causal relationships estimated without A/B testing

<strong>Statistical Significance:</strong>
- AI tool referrals (23 sessions) = small sample
- 6.5 weeks insufficient for weekly pattern detection
- Insufficient portfolio for language-specific content performance comparison

### 6.2 Lessons Learned

<strong>1. SEO is King</strong>
> 10〜20 hours SEO optimization = 2x daily users

SEO alone had greater impact than all other improvements (social automation, recommendation system, analytics) combined.

<strong>2. Don't Trust Traffic Spikes</strong>
> Average lies. Trust the median.

16.7 DAU (mean) vs. 12〜15 DAU (median) = 20〜40% difference. Nearly overestimated targets due to spike.

<strong>3. Language Strategy is About Competitive Advantage, Not Market Size</strong>
> English market is 100x bigger, but Korean is 100x easier.

Attracted 750 visitors with Korean content but English has 91% bounce rate. Fit matters, not size.

<strong>4. AI Tool Recommendations = Quality Certificate</strong>
> Perplexity, ChatGPT recommendations = Trust signal impossible to manipulate via SEO

Organic search can be gamed, but AI recommendations only acknowledge true content quality.

<strong>5. Returning Visitors Are the Real Asset</strong>
> 101 returning visitors > 649 new visitors

Returning visitors have 37% higher engagement. Retaining them is 5〜10x cheaper than acquiring new.

<strong>6. Data Quality > Data Quantity</strong>
> If 19.2% China traffic is bots, entire strategy was wrong.

Don't get drunk on 750 visitors - verify how many are actual humans.

<strong>7. Do Quick Wins First</strong>
> English homepage 91% bounce rate = serious problem fixable in 2 days

Wasting 125 people/month. Fix UX bugs before developing complex strategies.
