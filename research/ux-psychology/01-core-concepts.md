## 1. 인지 및 지각 관련

### 1.1 인지 부하 (Cognitive Load)
**정의**: 사용자가 정보를 처리하고 이해할 때 소요되는 정신적 에너지량

**작동 원리**:
- 워킹메모리(단기기억)의 한계에 기반
- 정보 제시나 작업이 복잡할수록 인지부하 증가
- 높은 인지부하 → 이해 어려움, 작업 완료 시간 증가

**적용 방법**:
- 폼을 여러 페이지로 분할
- 항목을 카테고리별로 그룹화
- 불필요한 항목 제거

**프론트엔드 구현**:
```css
/* 정보 그룹화를 위한 시각적 구분 */
.form-section {
  padding: 1.5rem;
  margin-bottom: 1rem;
  border: 1px solid #e2e8f0;
  border-radius: 0.5rem;
}
```

---

### 1.2 선택적 주의 (Selective Attention)
**정의**: 환경의 특정 자극에만 집중하고 나머지는 무시하는 과정

**UX 적용**:
- 중요 요소 강조
- 불필요한 시각적 잡음 제거
- CTA 버튼 명확하게 구분

---

### 1.3 배너 무시 현상 (Banner Blindness)
**정의**: 웹사이트의 광고 배너를 무의식적으로 무시하는 현상

**발생 원인**:
- 반복적 광고 노출로 면역 형성
- 선택적 주의의 일종

**대응 방안**:
- 네이티브 콘텐츠 형식 활용
- 사용자 목적과 관련성 높은 콘텐츠 배치
- 광고가 아닌 것처럼 보이는 형식 지양 (사용자 신뢰 저하 방지)

## 2. 의사결정 관련

### 2.1 앵커 효과 (Anchor Effect)
**정의**: 처음 제시된 정보를 기준점으로 삼아 이후 정보를 평가하는 현상

**적용 사례**:
```html
<!-- 가격 앵커링 -->
<div class="pricing">
  <span class="original-price">$199</span>
  <span class="sale-price">$99</span>
  <span class="discount">50% OFF</span>
</div>
```

**주의점**: 과도한 앵커링은 사용자 신뢰 저하 유발

---

### 2.2 미끼 효과 (Decoy Effect)
**정의**: 선택지에 미끼를 추가해 다른 옵션을 매력적으로 보이게 하는 기법

**Apple iPhone 사례**:
- iPhone 11: $699 (저가)
- iPhone 11 Pro: $999 (미끼)
- iPhone 11 Pro Max: $1,099 (고성능)

**핵심**: 상대적 비교를 통한 선택 유도

---

### 2.3 기본값 편향 (Default Bias)
**정의**: 변경할 유인이 없으면 제시된 기본값을 유지하는 현상

**적용 방법**:
```html
<!-- 사용자 이익 기반 기본값 설정 -->
<input type="checkbox" checked id="newsletter">
<label for="newsletter">뉴스레터 구독 (유용한 정보 제공)</label>
```

**윤리적 고려**: 사용자 이익을 우선하는 기본값 설정

---

### 2.4 결정 피로 (Decision Fatigue)
**정의**: 반복적 의사결정으로 합리적 선택이 어려워지는 현상

**해결 방법**:
- 선택지 수 제한 (힉의 법칙과 연계)
- 단계별 공개 (Progressive Disclosure)
- 추천 옵션 제시

---

### 2.5 확증 편향 (Confirmation Bias)
**정의**: 기존 신념과 맞는 정보만 수용하려는 경향

**UX 적용**:
- 다양한 관점의 정보 제공
- 사용자 테스트에서 편향 인식
- A/B 테스트로 가설 검증

---

## 3. 동기 부여 관련

### 3.1 손실 회피 (Loss Aversion)
**정의**: 이득보다 손실을 피하려는 심리가 더 강하게 작동

**HULU 사례**:
- 해지 절차 중 시청 불가능해질 콘텐츠 표시
- 손실감을 통한 해지 저지

**적용 예시**:
```html
<div class="warning-message">
  <span class="icon">⚠️</span>
  <p>지금 구매하지 않으면 이 가격은 내일 사라집니다</p>
</div>
```

---

### 3.2 희소성 효과 (Scarcity)
**정의**: 제한된 것에 더 높은 가치를 부여하는 현상

**Airbnb 사례**:
- 희소성 정보를 제목 바로 아래 표시
- "남은 방 3개" 등 긴급성 강조

**구현**:
```html
<span class="scarcity-badge">
  🔥 단 5개 남음
</span>
```

---

### 3.3 게이미피케이션 (Gamification)
**정의**: 게임 요소를 통해 사용자 참여와 동기 부여 향상

**Duolingo 사례**:
- XP 시스템: 경험치 획득 및 레벨 업
- 스트릭: 연속 사용일수 기록
- 리그: 경쟁 요소

**주요 요소**:
- 포인트 및 배지
- 랭킹 시스템
- 진행도 시각화
- 보상 메커니즘

---

### 3.4 변동형 보상 (Variable Reward)
**정의**: 예측 불가능한 보상으로 지속적 관심 유도

**작동 원리**: 도파민 분비 촉진으로 기대감과 흥분 증가

**TikTok 사례**:
- 알고리즘으로 다음 영상 예측 불가
- 무한 스크롤과 결합

---

### 3.5 목표 기울기 효과 (Goal Gradient Effect)
**정의**: 목표에 가까워질수록 행동과 노력 가속화

**구현**:
```html
<div class="progress-bar">
  <div class="progress" style="width: 80%"></div>
  <span class="progress-text">80% 완료! 거의 다 왔습니다!</span>
</div>
```

---

### 3.6 자이가르닉 효과 (Zeigarnik Effect)
**정의**: 미완료 과제가 완료된 과제보다 기억에 더 잘 남음

**Duolingo/Opal 사례**:
- 체크리스트 형식의 진행 표시
- 미완료 항목 시각적 강조

---

## 4. 사용자 경험 최적화

### 4.1 도허티 임계값 (Doherty Threshold)
**정의**: 0.4초 이내 응답 시 경험이 매끄럽게 느껴짐

**Facebook 사례**:
- 스켈레톤 화면으로 로딩 중 이탈 방지

**구현**:
```jsx
// 스켈레톤 로딩
const SkeletonCard = () => (
  <div className="animate-pulse">
    <div className="h-4 bg-gray-200 rounded w-3/4 mb-2"></div>
    <div className="h-4 bg-gray-200 rounded w-1/2"></div>
  </div>
);
```

---

### 4.2 노동 착각 (Labor Illusion)
**정의**: 처리 과정을 보여주면 결과의 가치를 높게 평가

**TurboTax/Netflix 사례**:
- 의도적으로 긴 로딩 화면 표시
- "당신에게 맞는 작품을 선택 중입니다" 메시지

**구현**:
```jsx
<div className="processing-screen">
  <Spinner />
  <p>최적의 결과를 찾고 있습니다...</p>
  <ul className="processing-steps">
    <li className="completed">✓ 데이터 분석 완료</li>
    <li className="in-progress">→ 개인화 적용 중...</li>
    <li className="pending">○ 결과 정리</li>
  </ul>
</div>
```

---

### 4.3 피크-엔드 법칙 (Peak-End Rule)
**정의**: 경험의 최고점과 끝점이 전체 평가를 좌우

**Mailchimp 사례**:
- 뉴스레터 발송 완료 시 축하 일러스트
- 감정적 피크 순간 설계

**적용 포인트**:
1. 최고의 순간 설계에 집중
2. 경험 종료 방식 최적화
3. 마지막 인상 긍정적으로 마무리

---

### 4.4 미적 사용성 효과 (Aesthetic-Usability Effect)
**정의**: 아름다운 디자인이 더 사용하기 쉽게 느껴짐

**핵심 요소**:
- 시각적 계층 구조
- 색상 및 타이포그래피 최적화
- 일관된 브랜드 미학
- 기능성과 아름다움의 균형

---

### 4.5 사용자 기쁨 (User Delight)
**정의**: 예상을 초과하는 경험에서 느끼는 기쁨과 놀라움

**Apple iOS16 사례**:
- 잠금화면 시계 폰트 변경 기능
- 기본 기능 동일, 심미성 향상

**MailChimp 사례**:
- 이메일 발송 전 원숭이 손 애니메이션
- 기능적 + 감정적 가치 제공

---

## 5. 정보 구조화

### 5.1 시각적 계층 (Visual Hierarchy)
**정의**: 디자인 내 중요 정보를 강조하는 기법

**핵심 요소**:
- 크기: 중요한 것을 크게
- 색상: 대비로 강조
- 위치: F형/Z형 패턴 활용
- 여백: 그룹화 및 분리

**Notion 랜딩페이지 사례**:
- 큰 색상 버튼으로 CTA 유도
- 제목/부제/본문 크기 차등화

---

### 5.2 단계적 공개 (Progressive Disclosure)
**정의**: 정보를 단계적으로 표시하여 필요 시 접근 가능하게 함

**Nike Japan 사례**:
- 기본 카테고리만 먼저 표시
- 호버 시 서브카테고리 드롭다운

**구현**:
```jsx
const Accordion = ({ title, children }) => {
  const [isOpen, setIsOpen] = useState(false);
  return (
    <div>
      <button onClick={() => setIsOpen(!isOpen)}>{title}</button>
      {isOpen && <div>{children}</div>}
    </div>
  );
};
```

---

### 5.3 계열 위치 효과 (Serial Position Effect)
**정의**: 첫 번째(초두)와 마지막(최근) 항목이 기억에 잘 남음

**Spotify 사례**:
- 하단 메뉴바 양 끝에 주요 기능 배치
- 항목을 3개로 제한

**적용 규칙**:
- 중요 기능을 목록 양 끝에 배치
- 항목 3-5개로 제한

---

## 6. 신뢰 및 설득

### 6.1 사회적 증명 (Social Proof)
**정의**: 다른 사람의 의견이나 행동에 영향 받는 현상

**B2B 서비스 사례**:
- 고객사 로고 상단 배치
- 사용자 리뷰 및 평점 표시

**구현**:
```html
<section class="social-proof">
  <h3>10,000+ 기업이 신뢰합니다</h3>
  <div class="client-logos">
    <img src="logo1.svg" alt="Client 1">
    <img src="logo2.svg" alt="Client 2">
  </div>
</section>
```

---

### 6.2 후광 효과 (Halo Effect)
**정의**: 한 특성의 긍정적 인상이 다른 특성 평가에 영향

**와인 실험 사례**:
- NPO 기부 정보 → 와인 맛 평가 향상

**UX 적용**:
- 기업의 사회적 책임 강조
- 신뢰성 있는 인증 마크 표시

---

### 6.3 부여 효과 (Endowment Effect)
**정의**: 소유한 것을 과도하게 평가하는 경향

**Notion 사례**:
- 초기 설정에서 맞춤형 템플릿 제공
- "자신만의 것"이라는 소유감 형성

---

### 6.4 매몰비용 효과 (Sunk Cost Effect)
**정의**: 과거 투자를 정당화하기 위해 계속 투자하는 경향

**교육 앱 사례**:
- 연간 선결제로 수강 완료율 상승
- 투자 회수 동기 활용

---

## 7. 사용자 행동 유도

### 7.1 호기심 간극 (Curiosity Gap)
**정의**: 정보 부족을 채우려는 흥미 활용

**Tinder 사례**:
- 좋아요한 사람 이미지 흐릿하게 표시
- 호기심 → 유료 결제 유도

---

### 7.2 의도적 마찰 (Intentional Friction)
**정의**: 의도적으로 행동을 방해해 신중한 결정 유도

**긍정적 사용**:
- 삭제 전 확인 모달
- 중요 작업 재확인

**부정적 사용 (지양)**:
- 복잡한 구독 취소 절차

---

### 7.3 스큐어모피즘 (Skeuomorphism)
**정의**: 디지털 인터페이스가 현실 물체를 모방

**Apple 계산기 사례**:
- 실제 계산기 버튼 배치 모방
- 클릭음으로 물리적 피드백 재현

**적용 시점**: 초기 학습 단계의 보조 수단으로 유용

---
