## 개선 3: 재시도 로직 추가 (web-researcher)

### 실행 전 상태

**문제점**:
- Brave Search API 실패 시 전체 리서치 실패
- 일시적 네트워크 오류에 취약
- 부분 실패 처리 없음
- 안정성: 95%

**영향**:
- 리서치 실패 시 수동 재실행 필요
- 사용자 경험 저하
- 블로그 작성 워크플로우 중단

---

### 실행 내용

**파일 수정**: `.claude/agents/web-researcher.md`

**추가된 섹션**: "Error Handling and Retry Logic" (Phase 2 내)

**구현 내용**:

#### 1. Automatic Retry (최대 3회)

```
Attempt 1: brave_web_search "[query]"
→ 실패 시: sleep 5 (더 긴 지연)

Attempt 2: brave_web_search "[query]"
→ 실패 시: sleep 10 (Exponential Backoff)

Attempt 3: brave_web_search "[query]"
→ 실패 시: 에러 보고 및 다음 검색 계속
```

#### 2. Exponential Backoff

- 1st retry: 5초
- 2nd retry: 10초
- 3rd retry: 스킵 및 계속

#### 3. Partial Success Handling

- 사용 가능한 결과로 계속 진행
- 실패한 검색 명확히 표시
- 수동 검증 제안

#### 4. Error Reporting

```markdown
⚠️ Search Failure Notice:
- Failed Query: "[query]"
- Attempts: 3
- Last Error: [error message]
- Recommendation: Manual search or retry later
```

---

### 실행 후 상태

**개선 사항**:
- ✅ 자동 재시도 (최대 3회)
- ✅ Exponential Backoff (5초 → 10초)
- ✅ 부분 실패 허용 (리서치 계속 진행)
- ✅ 명확한 에러 보고

**시나리오 2**: API Rate Limit 초과

- **Before**: 즉시 실패
- **After**: Exponential Backoff (5초 → 10초) → 성공
- **개선**: 대부분 자동 복구

**시나리오 3**: 일부 검색 실패

- **Before**: 전체 리서치 중단
- **After**: 부분 성공으로 계속, 실패한 검색만 표시
- **개선**: 리서치 완료 가능
