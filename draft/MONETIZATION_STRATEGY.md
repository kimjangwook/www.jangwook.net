## 1. 수익화 개요

### 1.1 AdMob 연동 성공

```
┌─────────────────────────────────────────────────────────────┐
│                    🎉 주요 변경사항                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  이전: Tauri + AdMob 네이티브 SDK 충돌 → AppLixir 사용      │
│                                                             │
│  현재: Tauri 커스텀 플러그인 개발로 AdMob 직접 연동 성공    │
│        - tauri-plugin-admob (Swift + Rust 브릿지)           │
│        - 보상형 광고 구현 완료                               │
│        - 전면/배너 광고 추가 가능                            │
│                                                             │
│  영향:                                                      │
│  ✅ 더 높은 eCPM (AdMob > AppLixir)                        │
│  ✅ 더 좋은 Fill Rate                                       │
│  ✅ 전면/배너 광고 사용 가능                                │
│  ✅ "광고 제거" IAP 가치 상승                               │
│  ✅ Mediation으로 수익 최적화 가능                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 수익화 3대 원칙

```
┌─────────────────────────────────────────────────────────────┐
│                    수익화 3대 원칙                           │
├─────────────────────────────────────────────────────────────┤
│  1. 게임 밸런스 보호: 과금은 외형만, 능력치 판매 금지        │
│  2. 무과금 유저 존중: 모든 콘텐츠 무료 획득 가능             │
│  3. 가치 제공 우선: 유저가 "쓸 가치가 있다"고 느끼는 상품    │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 수익 모델 비율 목표

| 수익원 | 목표 비율 | 설명 |
|--------|-----------|------|
| **광고 (보상형)** | 50% | 컨티뉴 (1차 도입) |
| **광고 (전면)** | 10% | 10게임마다 표시 (추후 도입) |
| **광고 제거 IAP** | 15% | 전면 광고 피로도 → 구매 유도 (추후) |
| **코인 패키지** | 15% | 반복 구매 유도 (추후) |
| **프리미엄 번들** | 10% | 고가 일회성 (추후) |

## 2. 광고 수익 전략

### 2.1 AdMob 광고 유형별 전략

```
┌─────────────────────────────────────────────────────────────┐
│                    1차 도입 (현재)                           │
├─────────────────────────────────────────────────────────────┤

│  게임 오버 후:                                               │

│  └─────────────────────────────────────────────────┘       │
│                                                             │
│  • 1차에서는 컨티뉴 광고만 도입                              │
│  • 유저 반응 및 광고 수익 데이터 수집 후 확장 결정          │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    2차 도입 예정 (추후)                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  게임 오버 후:                                               │
│  ┌─────────────────────────────────────────────────┐       │
│  │  "광고를 보고 코인 2배" → 획득 코인 × 2         │ 🔲    │
│  └─────────────────────────────────────────────────┘       │
│                                                             │
│  메인 메뉴:                                                  │
│  ┌─────────────────────────────────────────────────┐       │
│  │  "무료 코인 받기" → 50코인 (일 3회)             │ 🔲    │
│  │  "행운의 상자" → 랜덤 보상 (일 1회)             │ 🔲    │
│  └─────────────────────────────────────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| 보상형 광고 | 보상 | 일일 제한 | 예상 시청률 | 도입 시기 |
|-------------|------|-----------|-------------|-----------|
| 이어하기 | 부활 1회 | 게임당 1회 | 30-40% | ✅ 1차 |
| 코인 2배 | 코인 × 2 | 무제한 | 50-60% | 🔲 2차 |
| 무료 코인 | 50코인 | 3회 | 70-80% | 🔲 2차 |
| 행운의 상자 | 랜덤 | 1회 | 80-90% | 🔲 2차 |

## 5. 과금 전환 퍼널

### 5.1 유저 여정 설계

```
Day 0: 첫 설치
├── 튜토리얼 (쉬운 난이도)
├── 첫 게임 완료 → 첫 번째 챌린지 완료
├── 광고 노출 없음 (신규 보호)
└── 목표: "재미있다" 느끼게

Day 1-3: 습관 형성
├── 일일 챌린지 소개
├── 연속 로그인 보상 안내
├── 첫 전면 광고 노출 (Day 2, 4번째 게임)
├── 스타터 팩 배너 표시
└── 목표: "매일 하고 싶다"

Day 4-7: 과금 노출
├── 전면 광고 피로도 누적
```
